# Development Plan

## Phase 0: Documentation And Setup
Create docs, install dependencies, configure project, prepare route groups, and create placeholder folders.

Deliverables:
- Documentation files.
- Expo, TypeScript, Expo Router, NativeWind, and Tailwind setup.
- App config, EAS placeholder, and folder skeleton.
- Mock data only.

## Phase 1: UI Foundation
Build foundational components and tokens.

Deliverables:
- `SafeScreen`
- `AppCard`
- `PrimaryButton`
- `SecondaryButton`
- `IconButton`
- Typography helpers
- Color and spacing usage patterns

## Phase 2: Routing Foundation
Create Expo Router route groups and placeholder screens.

Deliverables:
- Onboarding routes.
- Tab routes.
- Modal routes.
- Navigation behavior.
- No backend or auth.

## Phase 3: Onboarding
Build Splash, Welcome, Intro, Goals, and Cycle Setup.

Deliverables:
- Calm onboarding flow.
- Goal selection.
- Basic cycle setup with safe copy.
- Progress dots.

## Phase 4: Main Tabs
Build Home, Cycle, Vault, and Profile.

Deliverables:
- Home daily summary.
- Cycle view with phase context.
- Vault record list using mock data.
- Profile settings and preferences.

## Phase 5: Modals
Build Bloop, Log Day, and Upload Record.

Deliverables:
- Bloop modal or overlay.
- Quick log form.
- Upload record flow with placeholder data.

## Phase 6: Polish
Improve animations, accessibility, responsiveness, and performance.

Deliverables:
- Accessibility labels.
- Keyboard behavior.
- Empty states.
- Loading states.
- Gentle motion.
- Performance review for lists and charts.

## Phase 7: App Store Preparation
Prepare release assets and store requirements.

Deliverables:
- EAS build configuration.
- App icons.
- Splash assets.
- Privacy copy.
- Store listing draft.
- Final accessibility and copy review.
